# Feature Specification

## アプリ概要

Mermaid Reviewer は、Markdown に埋め込まれた Mermaid 図や直接入力した Mermaid
定義を、デスクトップアプリ上でレビューするためのツールです。

## レイアウト

### 左ペイン

画面幅の約 1/3 を使用します。タブで `Files` と `Write` を切り替えます。

#### Files タブ

- Markdown ファイルを選択できる
- 選択された `.md` ファイルを読み込む
- 一度選択したファイルをボタン下部の履歴一覧から再度開ける
- 履歴一覧から不要なファイルを削除できる
- ファイル内の Mermaid コードブロックを抽出する
- 複数の Mermaid ブロックがある場合、一覧から選択して切り替えられる
- Mermaid ブロックがない場合は、その状態を明示する

対象コードブロック:

````md
```mermaid
graph TD
  A --> B
```
````

#### Write タブ

- Mermaid 定義を直接入力できる
- 入力内容はライブで右ペインに反映される
- 構文エラー時はプレビュー領域にエラーを表示する
- Files タブで選択した内容とは独立した編集状態を持つ

### 右ペイン

画面幅の約 2/3 を使用します。

- 現在選択中の Mermaid 定義を図として表示する
- Files タブでは選択された Mermaid ブロックを表示する
- Write タブでは編集中の Mermaid 定義を表示する
- ズーム操作に対応する
- ピンチ操作に対応する
- ドラッグによるパン操作に対応する
- 表示をリセットする操作を提供する

### 設定メニュー

タイトルセクション右側のスペースを使い、設定メニューを提供します。

- 初期実装ではカラースキーマを選択できる
- 選択されたスキーマはアプリ全体のパネル、ボタン、選択状態、プレビュー背景に反映する
- Mermaid 図のテーマ変数にも選択スキーマを反映する
- 選択されたカラースキーマは `~/.reviewers/settings.json`
  に保存し、次回起動時に復元する
- 今後、LLM設定や表示設定を同じメニューへ追加できる構造にする

## Mermaid 抽出仕様

- Markdown 内の fenced code block を対象にする
- info string が `mermaid` のコードブロックを抽出する
- 大文字小文字や空白をどこまで許容するかは実装時に決定する
- 抽出結果には以下を持たせる
  - 表示名
  - ファイル内の出現順
  - 開始行
  - Mermaid ソース本文

## 状態管理

初期実装では Preact のローカル state
を基本にします。状態が複雑化した場合のみ、軽量な状態管理ライブラリの導入を検討します。

主な状態:

- アクティブタブ
- 選択中の Markdown ファイル
- 抽出された Mermaid ブロック一覧
- 選択中の Mermaid ブロック
- Write タブの Mermaid 入力内容
- レンダリングエラー
- ズーム / パン状態
- 保存済み設定

## 設定保存

デスクトップアプリでは、設定をローカルファイルに保存します。

- 保存先: `~/.reviewers/settings.json`
- 保存項目: `colorScheme`、`summary`
- 起動時に設定を読み込み、UIへ反映する
- 設定ファイルが存在しない、または壊れている場合はデフォルト設定にフォールバックする
- `summary` は最近使った Markdown ファイルを最新順で保持する
- 同じパスを再度開いた場合は重複させず、`lastUsedAt` を更新する
- 履歴は最大5件とする
- 将来の設定項目は同じJSONへフィールドを追加する

例:

```json
{
  "colorScheme": "graphite",
  "summary": [
    {
      "path": "/Users/example/docs/spec.md",
      "lastUsedAt": "2026-07-18T01:23:45.000Z"
    }
  ]
}
```

## 将来拡張: LLM チャット

今後、レビュー支援用の LLM チャット領域を追加します。

想定仕様:

- Mermaid 図の内容を文脈として LLM に渡せる
- ローカル設定の API Key を利用する
- API Key は Deno 側で管理し、フロントエンドへ平文で渡さない
- 図の改善提案、構文修正、説明生成を依頼できる
- チャット履歴はセッション単位で保持する

この拡張を見越して、初期実装から Mermaid
ソース、抽出メタデータ、レンダリング状態を UI
コンポーネントに閉じ込めすぎない構成にします。

## 非目標

初期リリースでは以下を対象外にします。

- Markdown 全体の編集
- Mermaid 以外の図表コードブロック対応
- クラウド同期
- 複数ファイルの同時レビュー
- LLM チャットの本実装
